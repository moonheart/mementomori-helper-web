# 战斗日志系统设计文档

## 1. 概述

战斗日志系统用于记录和展示游戏中的战斗结果，包括角色输出、治疗、承伤等统计数据。

## 2. 数据模型

### 2.1 BattleResult / BattleSimulationResult 结构

```
BattleResult
├── battleTime: BattleTime           // 战斗开始/结束时间
├── questId: number                  // 关卡ID
├── reward: BattleReward             // 奖励信息
└── simulationResult: BattleSimulationResult
    ├── battleToken: string          // 唯一标识
    ├── battleEndInfo: BattleEndInfo // 战斗结果
    │   ├── isOutOfTurn: boolean     // 是否超时
    │   ├── endTurn: number          // 结束回合
    │   └── winGroupType: 1|2        // 1=攻击方胜, 2=防守方胜
    ├── battleField: BattleField     // 战场信息
    │   ├── battleType: BattleType   // 战斗类型
    │   ├── characters: BattleFieldCharacter[] // 角色详情
    │   └── damageBarList: DamageBar[]
    ├── battleLog: BattleLog         // 详细回合日志
    └── battleCharacterReports: BattleCharacterReport[] // 角色战报
```

### 2.2 BattleCharacterReport (角色战报)

```typescript
interface BattleCharacterReport {
  playerName: string;          // 玩家名称
  ownerPlayerId: number;       // 玩家ID
  groupType: 1 | 2;            // 1=攻击方, 2=防守方
  unitId: number;              // 角色ID
  unitType: UnitType;
  characterLevel: number;      // 等级
  characterRarityFlags: CharacterRarityFlags; // 稀有度
  elementType: ElementType;    // 属性
  
  // 战斗统计
  totalGiveDamage: number;     // 总输出伤害
  totalHpRecovery: number;     // 总治疗量
  totalReceiveDamage: number;  // 总承伤
  maxHp: number;               // 最大HP
  hp: number;                  // 剩余HP
}
```

## 3. 数据库设计

### 3.1 BattleLogEntity

```csharp
public class BattleLogEntity
{
    [Column(IsPrimary = true)]
    public string BattleToken { get; set; }        // 主键 - BattleToken
    
    public long UserId { get; set; }                // 用户ID
    public DateTime CreatedAt { get; set; }         // 记录时间
    
    // 战斗基本信息
    public int BattleType { get; set; }             // 战斗类型枚举
    public int? QuestId { get; set; }               // 关卡ID
    public bool IsWin { get; set; }                 // 是否胜利
    public int EndTurn { get; set; }                // 结束回合数
    public long BattleDurationMs { get; set; }      // 战斗时长(毫秒)
    
    // 来源信息
    public string ApiUri { get; set; }              // API 路径
    
    // 完整数据 (JSON 存储)
    [Column(StringLength = -1)]
    public string BattleDataJson { get; set; }
}
```

## 4. 后端实现方案

### 4.1 拦截保存机制

在 `OrtegaProxyController.ProxyRequest` 中拦截包含战斗结果的 API 响应，自动保存。

**拦截的 API 列表:**

| API URI | 响应类型 | 战斗数据路径 |
|---------|----------|-------------|
| `battle/boss` | `BattlebossResponse` | `battleResult` |
| `battle/pvpStart` | `BattlepvpStartResponse` | `battleResult` |
| `battle/legendLeagueStart` | `BattlelegendLeagueStartResponse` | `battleResult` |
| `towerBattle/start` | `TowerBattlestartResponse` | `battleResult` |
| `dungeonBattle/execBattle` | `DungeonBattleexecBattleResponse` | `battleSimulationResult` |
| `guildRaid/startGuildRaid` | `GuildRaidstartGuildRaidResponse` | `battleResult` |
| `guildTower/battle` | `GuildTowerbattleResponse` | `battleResult` |

### 4.2 服务层接口

```csharp
public class BattleLogService
{
    // 尝试保存战斗日志 (失败不影响主流程)
    public async Task TrySaveBattleLogAsync(long userId, string apiUri, object response);
    
    // 查询用户战斗日志
    public async Task<List<BattleLogEntity>> GetUserBattleLogsAsync(
        long userId, int? battleType, int page, int pageSize);
    
    // 根据 Token 获取详情
    public async Task<BattleLogEntity> GetByTokenAsync(string battleToken);
    
    // 删除单条记录
    public async Task<bool> DeleteAsync(long userId, string battleToken);
}
```

## 7. 数据保留策略

- 最近 30 天：保留完整数据
- 超过 30 天：自动清理 (后台任务)

## 8. 注意事项

1. **BattleToken 是唯一标识**，使用它作为主键
2. **保存失败不影响主流程**，使用 try-catch 包裹
3. **只保存关键战斗**，自动战斗不保存
4. **JSON 压缩存储**，减少数据库体积
5. **实时计算统计**，不存储冗余字段
